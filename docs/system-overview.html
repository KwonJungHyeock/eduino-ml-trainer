EDUINO ML 시스템 구조 설명서

작업자가 "어디서 무엇이 일어나는가"를 한눈에 이해하기 위한 독립 프로젝트 정의 문서

한 줄 요약

EDUINO ML 트레이너는 서버 코드 0줄의 순수 프론트엔드 도구입니다. 모든 학습·추론은 사용자 브라우저에서 일어나며, 서버는 페이지 첫 로드 시 HTML 한 번 전송하는 게 전부입니다.

이 프로젝트는 이렇게 구성됩니다

1

서버 측 호스팅 방식 결정

본 도구는 정적 파일 1개만 서빙되면 충분합니다. 어떤 정적 호스팅도 가능하며, 본 프로젝트는 AWS 호환·Python 생태계·기존 web-tutor와의 스택 일관성을 고려해 FastAPI 정적 라우트 방식을 채택했습니다.

2

브라우저 UI 구축

HTML/CSS/JS 단일 파일로 4단계 워크플로우(① 클래스 정의 → ② 데이터 수집 → ③ 모델 학습 → ④ 실시간 추론)를 구현. 학습 종류 탭(이미지·텍스트·숫자·음성)을 확장 자리로 마련.

3

ML 기능 탑재

TensorFlow.js를 통해 MobileNet v2(사전학습 · frozen · 특징 추출)와 분류 헤드(직접 학습 · trainable)를 브라우저에 탑재. 학습 가능한 부분은 분류 헤드뿐(전이학습).

4

결과 — 모든 ML이 브라우저에서

학습·추론·데이터 보관 모두 브라우저에서 수행. 서버 역할은 페이지 첫 전송에 국한되며 이후 통신은 0회. 사용자 수가 늘어도 서버 부담이 비례 증가하지 않음.

큰 그림부터 봅니다. 회색 박스(서버)는 정적 파일 1개만 내보내고, 보라색 박스(브라우저) 안에서 모든 ML 연산이 일어납니다.

FIG.1 — 서버는 작게, 브라우저는 크게

색이 칠해진 두 박스의 차이가 핵심입니다 — MobileNet(청록, frozen)은 가중치가 고정이고, 분류 헤드(코랄, trainable)만 우리가 학습시킵니다. 이게 전이학습의 본질이며, 적은 데이터·짧은 시간으로 높은 정확도가 나오는 이유입니다.

SECTION 2

서버 호스팅 방식 — 왜 FastAPI인가

먼저 사실관계를 분명히 합니다.

EDUINO ML 트레이너 (이 도구)

서버 코드 0줄. 순수 정적 HTML 한 개. FastAPI · nginx · S3+CloudFront · GitHub Pages 어떤 정적 호스팅이든 동작합니다.

EDUINO WEB TUTOR (별도 프로젝트)

OpenAI 챗봇 호출, 커리큘럼 동적 로딩 등 서버 로직이 실제로 필요해서 Python + FastAPI를 채택. 이미 AWS에 떠있는 상태.

본 프로젝트가 FastAPI 정적 라우트 방식을 채택한 이유

  • AWS 호환성 — EC2·ECS·Lambda·Elastic Beanstalk 어디든 자연스럽게 배포
  • 스택 일관성 — 옆 프로젝트(web-tutor)와 같은 Python 환경이라 운영·배포가 통일됨
  • 통합 가능성 — 향후 학습 결과 통계 수집, API 호출 등 백엔드 로직이 필요해질 때 같은 서버에서 확장 가능
핵심 짚어두기. 본 도구는 "서버 기술과 무관한 정적 자산"입니다. FastAPI 채택은 이 프로젝트의 본질이 아니라 배포 시점의 효율적 선택이며, 동일 결과물이 다른 정적 호스팅에서도 그대로 동작합니다. 
# 채택안: FastAPI 정적 라우트
app.mount("/ml-trainer", StaticFiles(directory="ml-trainer", html=True))

# 대안 — 같은 결과

# Amplify:     repo 연동 → 자동 정적 배포

핵심 인사이트. 학생 100명이 동시에 써도 서버 부담이 거의 0입니다. HTML 30KB는 CDN에 캐시되고, 학습·추론은 각 학생 PC에서 일어나며, 데이터는 각 학생 브라우저 RAM에만 머뭅니다. 사용자 수가 늘어도 서버 비용이 비례 증가하지 않습니다.

저장소는?

실질적으로 저장소가 없습니다. 학습 이미지와 모델 가중치는 모두 브라우저 RAM에만 존재하고, 페이지를 새로고침하거나 탭을 닫으면 사라집니다.

의도된 설계

  • 학생 얼굴·개인 데이터가 서버로 안 감 → 개인정보 안전
  • 서버 부하 0, 동시 사용자 무제한
  • 별도 동의 절차 없이 교육에서 사용 가능

트레이드오프

  • 새로고침 시 다시 학습해야 함
  • 다른 기기에서 같은 학습 모델 사용 불가
  • 학습 결과 영구 보관 불가
SECTION 4

AWS 배포 옵션 비교

본 도구를 AWS에 올리는 방법은 여러 가지입니다. 어느 것을 선택해도 EC2 인스턴스나 RDS 없이 학습·추론이 동작합니다.

옵션구성EC2 / DB월 비용평가
S3 + CloudFront index.html을 S3에 업로드 후 CloudFront 배포 없음 (서버리스) ~ $1~3 신규 단독 배포 시 베스트
AWS Amplify S3+CloudFront 매니지드. git 연동 자동 배포 없음 ~ $1~5 배포 자동화 편함
기존 FastAPI 재활용 web-tutor에 정적 라우트 1줄 추가 web-tutor의 기존 EC2 재사용 추가 $0 현재 인계안 — web-tutor 통합 시 베스트
EC2 + nginx (신규) 작은 인스턴스 1대 띄움 EC2 1대 ~ $5~15 오버스펙, 비추천
어떤 옵션이든 HTTPS 필수. 카메라 API는 비보안 컨텍스트에서 차단됩니다. S3+CloudFront와 Amplify는 무료 HTTPS 제공, FastAPI 재활용 시 기존 web-tutor 인증서 그대로 활용.

왜 서버 비용이 이렇게 적은가?

  • HTML 30KB가 CloudFront에 캐싱돼서 사용자 PC로 직접 갑니다 (서버 매번 처리 안 함)
  • TF.js·MobileNet 라이브러리도 외부 CDN에서 가져옵니다 (우리 서버 부담 0)
  • 학습·추론은 각 학생의 PC에서 일어납니다 (우리 서버 부담 0)
  • 데이터·모델은 각 학생 브라우저 RAM에만 있습니다 (우리 서버 부담 0)

SECTION 7

그래서 왜 이렇게 단순한가?

네 가지가 맞물려서 가능합니다.

① 전이학습 (Transfer Learning)

MobileNet은 ImageNet의 1,400만 장으로 이미 학습된 모델입니다. "사물을 구분하는 기본 시각 능력"이 가중치 안에 들어있죠. 우리는 그 능력을 그대로 빌려와서, 마지막 분류 단계만 새로 학습합니다. 0부터 학습하면 클래스당 수천 장과 GPU 서버가 필요한데, 전이학습은 클래스당 20장이면 됩니다.

② MobileNet의 경량성

이 모델 자체가 모바일 폰에서 실시간 동작하도록 설계됐습니다. 4MB 정도의 가벼움이라 CDN에서 받아 브라우저에서 돌리는 게 부담 없습니다.

③ TensorFlow.js + WebGL 가속

브라우저가 GPU를 활용할 수 있어서, 사용자의 PC가 곧 학습 서버가 됩니다. 학생 100명이 동시에 써도 우리 서버에 부하가 0이에요 — 각자의 PC에서 학습하니까요.

④ 분류 헤드의 작음

약 1024차원 입력 → 100 유닛 → N 클래스. 학습할 파라미터가 약 10만 개 수준이라, 클래스당 20장으로 30 epoch 돌리면 일반 노트북에서 5~10초면 끝납니다.

요약. 무거운 부분(특징 추출)은 미리 학습된 모델로 떠넘기고, 가벼운 부분(분류 헤드)만 사용자 브라우저에서 그때그때 학습 — 이게 이 구조의 단순함이 오는 곳입니다. 텍스트·숫자·음성 분류로 확장할 때도 같은 패턴(전이학습용 frozen 인코더 + 작은 학습 헤드)이 그대로 적용됩니다.
SECTION 8

저장 요구가 추가되면 어떻게 되는가

현재는 "단발성 실습"이 목적이라 저장이 필요 없지만, 요구사항이 추가되는 순간 인프라 복잡도가 달라집니다.

저장 요구필요 인프라서버 부담복잡도
없음 (새로고침 시 소실) 현재 그대로 없음 낮음
같은 기기·브라우저 유지 브라우저 IndexedDB 없음 낮음
다른 기기에서도 학습 결과 사용 사용자 인증 + S3 또는 DynamoDB 중간 중
학생 학습 통계·진도 수집 백엔드 API + DB (RDS/DynamoDB) 중간 ~ 큼 중 ~ 높음

핵심은 — "저장 안 함"이라는 결정 자체가 인프라 비용·복잡도·개인정보 위험을 동시에 0에 가깝게 만드는 설계 선택이라는 점입니다. 향후 저장 요구가 생길 때는 위 표에서 가장 가벼운 단계(IndexedDB)부터 검토하는 게 권장됩니다.

SECTION 9

작업자 체크포인트

이 시스템을 다룰 때 가장 자주 헷갈리는 지점들입니다.

오해실제
"서버에 ML 모델을 띄워야 하나?" 아니오. 모델은 사용자 브라우저에서 동작합니다. 서버는 정적 파일만.
"학습 데이터를 DB에 저장해야 하나?" 아니오. 브라우저 RAM에만. 영구 저장 필요 시 IndexedDB부터 검토.
"EC2 / RDS 없으면 학습이 안 되나?" 됩니다. S3 + CloudFront만으로도 동작. 정적 파일만 사용자에게 도달하면 OK.
"서버 부하가 학생 수에 비례하나?" 아니오. HTML 1회 받은 뒤로 통신 없음. 비용은 동시 페이지 로드 수에만 비례하고 그것도 CDN 캐싱으로 미미.
"HTTPS 없어도 되나?" 안 됩니다. 카메라 API는 HTTPS에서만 동작합니다.
"GPU 서버가 필요한가?" 아니오. 사용자 PC의 GPU/CPU만 씁니다. 서버는 CPU 한 줌이면 충분.
"AWS 외 다른 환경에선?" 어디든 OK. 정적 파일 1개만 서빙되면 됩니다.
